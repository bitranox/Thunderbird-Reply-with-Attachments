ദ്രുതാരംഭം

ദ്രുതാരംഭം​

കുറഞ്ഞ Thunderbird പതിപ്പ്

ഈ ആഡ്‑ഓൺ Thunderbird 128 ESR അല്ലെങ്കിൽ അതിനേക്കാൾ പുതിയ പതിപ്പുകൾ പിന്തുണയ്ക്കുന്നു. പഴയ പതിപ്പുകൾ പിന്തുണയ്ക്കുന്നില്ല.

ടെലിമെട്രിയില്ല; പശ്ചാത്തല നെറ്റ്വർക്കില്ല

ആഡ്‑ഓൺ ഒന്നും അനാലിറ്റിക്‌സ്/ടെലിമെട്രി ശേഖരിക്കുന്നില്ല, കൂടാതെ ഒന്നും പശ്ചാത്തല നെറ്റ്വർക്ക് അഭ്യർത്ഥനകൾ അയക്കുന്നില്ല. നെറ്റ്വർക്കിലേക്ക് പ്രവേശനം നിങ്ങൾ ബാഹ്യ കണ്ണികൾ (Docs, GitHub, Donate) ക്ലിക്കുചെയ്യുമ്പോൾ മാത്രമേ ഉണ്ടാകൂ.

ഇൻസ്റ്റാൾ​

  1. Thunderbird Add‑ons നിന്നും ആഡ്‑ഓൺ ഇൻസ്റ്റാൾ ചെയ്യുക.
  2. ഐച്ഛികം: സ്ഥിരീകരണം സജീവമാക്കുക (Options → “അറ്റാച്ച്മെന്റുകൾ ചേർക്കുന്നതിനുമുമ്പ് ചോദിക്കുക”).
  3. ഐച്ഛികം: ബ്ലാക്ക്‌ലിസ്റ്റ് മുന്നറിയിപ്പ് (ഡീഫോൾട്ട്) പ്രവർത്തനക്ഷമമായി തന്നെ വിടുക: “ബ്ലാക്ക്‌ലിസ്റ്റ് മൂലം അറ്റാച്ച്മെന്റുകൾ ഒഴിവാക്കിയാൽ മുന്നറിയിപ്പ് നൽകുക”.
  4. ഐച്ഛികം: ബ്ലാക്ക്‌ലിസ്റ്റ് പാറ്റേണുകൾ ചേർക്കുക (ഓരോ വരിയിലും ഒന്നു വീതം), ഉദാ:
*intern*
*secret*
*passwor*  # matches both “password” and “Passwort” families

കുറിപ്പ്: മുകളിൽ കാണുന്ന “# …” ഈ ഡോക്യുമെന്റേഷനിലെ ഒരു കമന്റാണ്; Options-ിൽ നിങ്ങൾ പേസ്റ്റ് ചെയ്യുന്ന പാറ്റേണുകളിൽ കമന്റുകൾ ഉൾപ്പെടുത്തരുത്. ഓരോ വരിയിലും ഒരു പാറ്റേൺ മാത്രം നൽകുക.

ഇപ്പോൾ അറ്റാച്ച്മെന്റുകളുള്ള ഒരു സന്ദേശത്തിന് മറുപടി നൽകുക — യഥാർത്ഥ ഫയലുകൾ സ്വയമേവയോ അല്ലെങ്കിൽ വേഗത്തിലുള്ള സ്ഥിരീകരണത്തിനുശേഷമോ ചേർക്കപ്പെടും. നിങ്ങളുടെ ബ്ലാക്ക്‌ലിസ്റ്റ് കൊണ്ട് ഏതെങ്കിലും ഫയലുകൾ ഒഴിവാക്കപ്പെട്ടാൽ, അവയുടെ പട്ടികയോടുകൂടിയ ഒരു ചെറു മുന്നറിയിപ്പ് നിങ്ങൾക്ക് കാണാം.

സ്ഥിരീകരിക്കുക​

  • 1–2 അറ്റാച്ച്മെന്റുകളുള്ള ഒരു സന്ദേശത്തിന് മറുപടി നൽകുക, യഥാർത്ഥ ഫയലുകൾ നിങ്ങളുടെ compose വിൻഡോയിലേക്ക് ചേർത്തിട്ടുണ്ടെന്ന് സ്ഥിരീകരിക്കുക.
  • പ്രവർത്തനം ക്രമീകരിക്കാൻ ക്രമീകരണം കാണുക (സ്ഥിരീകരണ ടോഗിൾ, സ്വതേയുള്ള മറുപടി, ബ്ലാക്ക്‌ലിസ്റ്റ് പാറ്റേണുകൾ).

ബ്ലാക്ക്‌ലിസ്റ്റ് മുന്നറിയിപ്പ് സ്ഥിരീകരിക്കുക​

  • “secret.txt” പോലെയുള്ള ഒരു ഫയൽ ഉൾക്കൊള്ളുന്ന സന്ദേശത്തിന് മറുപടി നൽകുക.
  • “ബ്ലാക്ക്‌ലിസ്റ്റ് മൂലം അറ്റാച്ച്മെന്റുകൾ ഒഴിവാക്കിയാൽ മുന്നറിയിപ്പ് നൽകുക” പ്രവർത്തനക്ഷമമാണെങ്കിൽ, ഒരു ചെറിയ ഡയലോഗ് ഒഴിവാക്കപ്പെട്ട ഫയലുകളും പൊരുത്തപ്പെട്ട പാറ്റേണും പട്ടികപ്പെടുത്തും.

മുന്നറിയിപ്പ് കാണുന്നില്ലെങ്കിൽ, പാറ്റേൺ ഫയൽനാമവുമായി കൃത്യമായി പൊരുത്തപ്പെടുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക (ഫയൽനാമം മാത്രം, case‑insensitive). ക്രമീകരണം → Blacklist കാണുക.

കീബോർഡ് കുറിപ്പ്​

  • സ്ഥിരീകരണ ഡയലോഗിൽ Yes ന് Y/J, Noയ്ക്ക് N/Esc പിന്തുണയ്‌ക്കുന്നു. ചില non‑Latin കീബോർഡുകളിൽ അക്ഷര കീകൾ വ്യത്യാസപ്പെടാം; Enter അമർത്തുന്നത് ശ്രദ്ധ കേന്ദ്രീകരിച്ചിരിക്കുന്ന ബട്ടണിനെ സ്ഥിരീകരിക്കും.
